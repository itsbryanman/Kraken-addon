This file documents the fixes and changes made during the “3 user-reported bugs” pass.

  - Set `logo` to the raw GitHub icon URL (correct repo path):  
    `https://raw.githubusercontent.com/itsbryanman/Kraken-addon/main/kraken-addon/public/icon.png`
  - Removed the placeholder `contactEmail: 'kraken@example.com'`

### 2) Improve quality parsing (reduce “Unknown”)

Changes are implemented in the stream `name` formatting logic:

- Explicit quality mapping in `kraken-addon/src/ranking/scorer.ts`
  - If title contains `CAM`, `TS`, `HDCAM`, `HDTS`, `TELESYNC` → `"[CAM] Kraken"`
  - If title contains `2160p`, `2160`, `4k`, `uhd` → `"[4K] Kraken"`
  - If title contains `1080p` → `"[1080p] Kraken"`
  - If title contains `720p` → `"[720p] Kraken"`
  - Otherwise, fall back to `"[<resolution>] Kraken"` or `"[Unknown] Kraken"`
  - Keeps the debrid prefix tag when applicable (e.g. `"[REALDEBRID+]"`)
- Sorting update in `kraken-addon/src/ranking/scorer.ts`
  - Streams with `resolution === "Unknown"` are always pushed to the bottom of the list.

### 3) Diversify sources (timeouts + debug logs)

There was no `Promise.race` timeout in this codebase; instead, provider requests use Axios timeouts and the provider queue already had a 15s timeout configured.

- Increased provider HTTP timeouts from 10s → 15s in `kraken-addon/src/providers/base.ts`
  - Axios client default timeout is now 15s.
  - HTML scraper mirror fetch timeout is now 15s.
- Added visibility into scraper failures (printed to server logs)
  - YTS mirror failures: `kraken-addon/src/providers/tier1-primary.ts` (`console.log` includes HTTP status/message)
  - 1337x failures: `kraken-addon/src/providers/tier1-primary.ts`
    - Logs when magnet links aren’t found (likely block/parsing)
    - Logs when detail pages fail (HTTP status/message)
    - Logs when the overall search fails

### 4) Include filenames in results (UX improvement)

Implemented in `kraken-addon/src/handlers/stream.ts`:

- Stream `title` now starts with a filename line: `📄 <filename>`
  - Uses debrid-resolved filename when available
  - Otherwise uses the best video file from `torrent.files` (largest video) when present
  - Otherwise falls back to the torrent title
- `behaviorHints.filename` is now set even for non-debrid (P2P) streams.
